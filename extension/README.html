What it does

OpenEvidence protects its API with bot detection that blocks plain server requests. This extension removes that problem: when an AI tool (Claude Code, Codex, …) asks OpenEvidence a question through the openevidence-mcp server, the request is run inside your own logged-in OpenEvidence tab — so it carries your genuine browser session and is never challenged. The extension is a generic authenticated fetch proxy; all the OpenEvidence logic stays in the local MCP server.

Nothing navigates or pops up — the tab stays where it is. The extension only ever talks to openevidence.com and your local relay.

Set it up (once)

  1. Keep this extension loaded in one Chromium browser (Chrome, Edge, Brave, Arc, …).
  2. Stay logged in to openevidence.com in that browser, and keep a tab open. That login is your authentication — no API key, no cookie file.
  3. Run your AI tool. The openevidence-mcp server auto-starts the relay and connects to this extension. The badge above turns green.
One tab, many sessions. The relay runs as a shared background daemon, so you can run OpenEvidence from any number of Claude/Codex sessions at the same time — they all flow through this single logged-in tab. No per-session setup.

Privacy & security

  • The relay listens on localhost only (127.0.0.1:__RELAY_PORT__) — nothing is exposed to the network.
  • The extension stores no credentials. Your OpenEvidence session lives in your browser, as always.
  • It only acts on requests from your own local MCP server, and only against openevidence.com.

Troubleshooting

  • Badge not green? The debug panel above lists the exact failure and the fix. This page also rechecks every 3 seconds, so it turns green by itself once the cause is fixed.
  • Reloading the extension: chrome://extensions (or brave://extensions, edge://extensions) → ↻ on this extension. Needed after every rebuild.
  • Changed OE_MCP_RELAY_PORT? Rebuild this extension with the same value (cd extension && OE_MCP_RELAY_PORT=<port> npm run build) and reload it — the port is baked in at build time. npm run doctor checks the sync.
  • Run the relay only in one browser at a time if you have it installed in several — requests go to whichever answers first.
